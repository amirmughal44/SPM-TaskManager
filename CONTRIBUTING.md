# Contributing Guidelines

Thank you for contributing to SPM Task Manager. This project follows a simple team workflow designed for the SPM lab.

## Branching Rules

- `main` stores stable release-ready code.
- `dev` stores integrated development work.
- Feature work must happen on feature branches such as `feature-task-core`, `feature-cli`, or `feature-docs`.
- Feature branches must be merged into `dev` before `dev` is merged into `main`.

## Commit Rules

- Make small, focused commits.
- Use meaningful commit messages.
- Reference issue numbers in commit messages when possible.
- Run tests before requesting review.

## Pull Request Rules

- Open pull requests against `dev`.
- Include a short summary and test notes.
- Request review from another team member.
- Merge only after review approval.

## Code Review Rules

- Review for correctness, readability, tests, and maintainability.
- Leave at least one useful review comment.
- Resolve review comments before merging.

## Conflict Resolution

- Resolve conflicts manually.
- Keep both branches' useful changes where possible.
- Commit the resolved version with a clear message.
